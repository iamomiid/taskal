# Security Policy

## Reporting a Vulnerability

Please do not open public GitHub issues for security vulnerabilities.

Instead, report them privately through GitHub security advisories or by contacting the maintainer directly if private reporting is not yet configured.

When reporting a vulnerability, please include:

- affected action name and version
- runner environment details
- steps to reproduce
- expected impact

I will do my best to acknowledge reports promptly and work toward a fix.

## Supported Versions

Security support is best-effort for the latest state of the `main` branch until tagged releases are established.
